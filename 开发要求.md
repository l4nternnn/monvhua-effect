# 任务：开发 Minecraft 1.21.8 Fabric Mod「魔女化状态效果」

## 项目基础信息

- **Minecraft 版本**：1.21.8
- **Mod Loader**：Fabric
- **Mod ID**：`monvhua`
- **包名**：`com.kuilunfuzhe.monvhua`
- **Java 版本**：21
- **Mappings**：Mojang Official Mappings（不是 Yarn）
- **项目结构**：已勾选 Split client and common sources，分为 `src/main/` 和 `src/client/` 两个源集
- **依赖**：Fabric API

项目模板已通过 https://fabricmc.net/develop/template/ 生成，你需要在已有模板基础上添加/修改代码。

## 功能需求总览

实现一个**纯显示用途**的自定义状态效果（StatusEffect）系统：

- 玩家按 E 打开背包时，右侧的药水效果列表中会显示当前所处的「魔女化阶段」对应的状态效果图标和名称（与原版药水效果并列显示）
- 状态效果**仅作显示用途**，不附加任何 buff/debuff，不修改玩家任何属性
- 状态效果的种类（共 8 个）由名为 `monvhua` 的计分板（dummy 类型，由外部数据包维护）数值决定
- 当玩家计分板数值变化、跨越阈值时，自动切换为对应阶段的状态效果
- 同时，阶段变化时通过聊天栏推送该阶段的完整描述文本

## 核心设计

### 1. 状态效果机制

- 注册 **8 个独立的自定义 StatusEffect**，每个对应一个魔女化阶段
- 类别（StatusEffectCategory）：根据阶段倾向选择
  - 神智清醒、略染污浊：`BENEFICIAL`（绿色边框）
  - 轻度魔女化、中度魔女化：`NEUTRAL`（蓝色边框）
  - 高度魔女化及以上：`HARMFUL`（红色边框）
- 每个效果的 color 字段使用对应阶段的 RGB 颜色值（用于背包列表中效果图标背景色）
- 效果实现类继承 `StatusEffect`，**不重写 `applyUpdateEffect`**（或重写后直接 `return true` 不做任何事），保证零 buff/debuff
- 每个效果需要对应的 PNG 图标（32×32），路径 `assets/monvhua/textures/mob_effect/<阶段id>.png`，先用占位图也可以

### 2. 阶段切换逻辑（服务端）

- 每秒（每 20 tick）读取一次所有在线玩家的 `monvhua` 计分板数值
- 根据数值映射到 8 个阶段之一
- 与缓存中该玩家上一次的阶段比较：
  - **如果阶段变化**：
    1. 移除该玩家身上所有 8 个魔女化效果（无论之前是哪个）
    2. 添加新阶段对应的 StatusEffect，duration 设置为 `StatusEffectInstance.INFINITE`（即 -1，永久），amplifier 为 0，showParticles 为 false，showIcon 为 true
    3. 在聊天栏推送阶段名 + 完整描述（首次进入游戏初始化时不显示"阶段变化"字样）
  - **如果阶段未变化**：什么都不做（不刷新效果，不刷屏）
- 玩家断开连接时清理缓存

### 3. 关键细节

- 状态效果使用 `INFINITE` 持续时间，避免每秒刷新带来的"持续时间数字不停跳动"的视觉问题
- 由于是永久效果，**只在阶段变化时**调用 `addStatusEffect`，避免反复添加导致客户端闪烁
- 服务端添加效果会自动同步到客户端，不需要自定义网络包
- 玩家死亡重生时，原版会清除所有状态效果，下一秒服务端 tick 时缓存比对会发现"无效果但分数对应某阶段"，需要重新添加 → 在缓存逻辑中处理：**死亡时也清理缓存**，让重生后能重新触发"添加 + 推送描述"

### 描述文本（用于阶段变化时聊天栏推送）

- **神智清醒**：初来乍到，对未来还有着希望，对魔法的了解仅限于知道如何释放。理性没有受到影响，仅能在日常探索中使用魔法辅助。
- **略染污浊**：逐渐熟悉这里，也感觉到了魔女因子对你的影响，一些古怪的念头冒出。
- **轻度魔女化**：意外频发，十分不安。理性逐渐开始受到侵扰，魔法开始波动。
- **中度魔女化**：思考不断受影响，纷乱的念头不断涌现。魔法可在任何地点或时间中使用。开始互相猜忌、怀疑，不能再轻易信任彼此。
- **高度魔女化**：亲眼见证血腥、暴力、背叛与冷漠之后，逐渐不受控制，难以从可怕的想法中挣扎。魔法获得强化，但代价是什么？
- **重度魔女化**：难以继续保持理性，黑暗想法几乎要控制你。状态开始波动，对躯体的掌控力下降，但魔法得到大幅加强。
- **准魔女**：理性所剩无几，杀人欲望强烈无比，是一座即将爆发的火山。杀人后会暂时冷静下来，但欲望终将再次降临。此阶段仍须遵守杀人限制，凶杀会大幅减少魔女化进度。
- **魔女**：最后的理智也即将丢失。没有救赎，没有希望，强烈的杀人欲望充斥脑海。并非完全失去意识，而是在极强杀人欲望驱使之下保留最后一点意志——需要进行RP演绎，而非无脑清图行为。杀完一个人后也会暂时冷静。被看守抓捕，也许是你不再伤害他人最后的方式。作为怪物，你已经没有回头路了。

阶段判定规则：取**不超过当前数值**的最大阈值对应的阶段。

## 具体实现要求

### 1. `DisplayOnlyEffect.java`

```
public class DisplayOnlyEffect extends StatusEffect {
    public DisplayOnlyEffect(StatusEffectCategory category, int color) {
        super(category, color);
    }
    // 不重写 applyUpdateEffect；如必要，重写并直接返回 true 不做任何事
    // 不重写 canApplyUpdateEffect
}
```

### 2. `WitchStage.java`（enum）

每个枚举常量持有：
- `int threshold`
- `String id`（用于 Identifier 和翻译键）
- `String displayName`
- `String description`
- `int color`
- `StatusEffectCategory category`
- `RegistryEntry<StatusEffect> effectEntry`（在 MonvhuaMod 注册时回填，使用静态方法或在 MonvhuaMod 中维护一个 `EnumMap<WitchStage, RegistryEntry<StatusEffect>>`）

提供静态方法：
- `public static WitchStage fromScore(int score)`：返回数值不超过 score 的最高阶段

### 3. `MonvhuaMod.java`（implements ModInitializer）

#### 字段
- `public static final String MOD_ID = "monvhua";`
- `public static final String OBJECTIVE_NAME = "monvhua";`
- `public static final EnumMap<WitchStage, RegistryEntry<StatusEffect>> EFFECTS = new EnumMap<>(WitchStage.class);`
- `private static final Map<UUID, WitchStage> lastStage = new HashMap<>();`

#### `onInitialize()`
1. **注册所有 8 个效果**：遍历 `WitchStage.values()`，对每个阶段：
   ```java
   RegistryEntry<StatusEffect> entry = Registry.registerReference(
       Registries.STATUS_EFFECT,
       Identifier.of(MOD_ID, stage.id),
       new DisplayOnlyEffect(stage.category, stage.color)
   );
   EFFECTS.put(stage, entry);
   ```

2. **注册服务端 tick 事件**（`ServerTickEvents.END_SERVER_TICK`）：
   - 每 20 tick 执行一次
   - 获取 scoreboard 和名为 `monvhua` 的 objective，objective 为 null 则跳过
   - 遍历所有在线玩家：
     - 通过 `scoreboard.getScore(player, objective)` 获取 `ReadableScoreboardScore`，null 时按 0 处理
     - 计算 `current = WitchStage.fromScore(value)`
     - 与 `lastStage.get(uuid)` 比对：
       - 如果不同：
         - 更新缓存
         - 移除玩家身上所有 8 个魔女化效果（遍历 EFFECTS values 调用 `player.removeStatusEffect(entry)`）
         - 添加新效果：`player.addStatusEffect(new StatusEffectInstance(EFFECTS.get(current), StatusEffectInstance.INFINITE, 0, false, false, true))`（参数：effect, duration, amplifier, ambient, showParticles, showIcon）
         - 推送聊天栏：
           - 如果 previous 不为 null：先发一条 `Text.literal("【阶段变化】").formatted(GRAY).append(Text.literal(current.displayName).formatted(对应颜色))`
           - 发一条 `Text.literal("◆ " + current.displayName).formatted(对应颜色, BOLD)`
           - 发一条 `Text.literal(current.description).formatted(GRAY)`
           - 都用 `player.sendMessage(text, false)`

3. **注册玩家断开事件**（`ServerPlayConnectionEvents.DISCONNECT`）：
   - 从 `lastStage` 移除该玩家 UUID

4. **注册玩家死亡事件**（用 `ServerLivingEntityEvents.AFTER_DEATH` 或 `ServerPlayerEvents.AFTER_RESPAWN`）：
   - 从 `lastStage` 移除该玩家 UUID（让下一秒重新触发添加效果 + 推送描述）

> 注意：阶段颜色到 `Formatting` 的映射可在 `WitchStage` 枚举中再加一个 `Formatting chatColor` 字段，分别对应：GREEN, DARK_GREEN, YELLOW, GOLD, RED, DARK_RED, LIGHT_PURPLE, DARK_PURPLE。

## API 注意点（1.21.8 Mojang Mappings）

- 注册效果：`Registry.registerReference(Registries.STATUS_EFFECT, Identifier.of(...), new DisplayOnlyEffect(...))` 返回 `RegistryEntry<StatusEffect>`
- 添加效果：`StatusEffectInstance` 构造器需要 `RegistryEntry<StatusEffect>` 而不是 `StatusEffect` 本身
- 永久持续时间常量：`StatusEffectInstance.INFINITE`（值为 -1）
- 计分板：`scoreboard.getScore(ScoreHolder, ScoreboardObjective)` 返回 `ReadableScoreboardScore`，需判空
- 玩家发消息：`player.sendMessage(Text, boolean)`，false 表示聊天栏（true 是 action bar）
- `Text.literal(...).formatted(Formatting...)` 可传多个 Formatting

## 验收标准

完成后，按以下步骤可以正常测试：

1. `./gradlew runClient` 启动客户端
2. 创建一个开作弊的单人世界
3. 执行：
   ```
   /scoreboard objectives add monvhua dummy 魔女化
   /scoreboard players set @s monvhua 0
   ```
4. **按 E 打开背包**，右侧应该出现一个绿色的「神智清醒」状态效果图标，鼠标悬停显示效果名，左下角持续时间显示为 `**:**`（无限）
5. 聊天栏应有一条该阶段的完整描述
6. 关闭背包，执行 `/scoreboard players set @s monvhua 45`：
   - 1 秒内，再次按 E 打开背包，右侧效果应已切换为金色的「中度魔女化」
   - 聊天栏出现「阶段变化」提示 + 该阶段描述
7. 多次切换数值（跨越和不跨越阈值），验证：
   - 跨越阈值：效果切换 + 聊天栏推送
   - 未跨越阈值：什么都不发生（不刷屏、不闪烁）
8. 玩家死亡后重生，应自动重新获得当前阶段的效果
